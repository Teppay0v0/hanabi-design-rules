# HANABI デザインルール

合同会社HANABI 廣田哲平が HP・LP・スライド制作で実践しているデザイン判断基準。
Claude Code が出力するデザインの質を上げるためのスキル。

## なぜこのスキルが必要か

Claude Code にデザインを任せると、デフォルトでは：
- 要素が詰まりがちになる
- カラフルで彩度が高くなる
- ドロップシャドウ・角丸・グラデーションが入る
- 絵文字がアイコン代わりになる
- ストック写真が使われる

これを防ぐための判断ルール集。**スターターテンプレートは `references/lp-starter.html`。白紙から書かない。**

---

## 根本哲学：引くこと

AIは「足す」のが得意。だから人が「引く」必要がある。

> 「AIが作るデザインって、なんか詰まってる感じがする。
> わたしが気をつけてるのは、むしろ引くこと。
> 余白って『何もない』んじゃなくて、ちゃんと機能してると思っていて。
> 詰め込みすぎると、見る人が疲れる。何を見たらいいか分からなくなる。
> だから意図的に『置かない』判断が大事。」

---

## 禁止事項（やらないこと）

以下は理由を問わず使用しない。

| 禁止 | 代わりに使うもの |
|------|----------------|
| 絵文字（すべての制作物） | モノライン系アイコン（ICON RAINBOW / ICONOON MONO） |
| イタリック ※1 | 使用しない。強調はサイズ・ウェイト・字間で |
| 高彩度のベタ塗り背景・全面カラー背景 | 白と極薄ティント（wash）を交互に |
| ドロップシャドウ ※2 | 細罫（1px solid var(--line)）のみ |
| 角丸カード（border-radius） | 直角のみ |
| グラデーション | 単色のみ |
| 主張する色を3色以上 | メイン＋サブの2色＋インク・グレー・罫のみ |
| 大型・角丸の CTA ボタン | テキストリンクのみ（下線とセットで。色だけでリンクと分からせない） |
| ヒーロー写真・ストック画像 | 使用しない（タイポグラフィ・線・数字で構成） |
| 太字＋カラー＋italic の複合強調 | ウェイト・サイズ・letter-spacing のどれか1つだけ |
| 機械的な 8/16/24px 余白倍数 | 読みやすさ優先で広く取る（ただし振れ幅には規則性を持たせる） |

> ※1 日本語フォントはイタリック体を持たないため、CSSで `font-style: italic` を当てても標準書体を機械的に傾けた斜体（oblique）になるだけで可読性が落ちる。デジタル庁デザインシステムも「そうしなければならない理由がある場合を除いて避ける」としている（[タイポグラフィ](https://design.digital.go.jp/dads/foundations/typography/)）。美意識だけでなく技術的に正しい判断。
> ※2 ドロップシャドウは単色ではないため、要素と背景のコントラスト比を保証できない。高さ（奥行き）を表現したい要素は、境界にボーダー（細罫）を持たせてコントラスト比3:1以上を確保する（[エレベーション](https://design.digital.go.jp/dads/foundations/elevation/)）。hanabiの「奥行きは細罫で表現する」はこの原理と一致している。

## カラー設計（メイン＋サブの2色）

配色は **メインカラー1色 ＋ サブカラー1色** で組む。3色目・4色目を足さない。
（参照：[2色配色の考え方 — design-get.com](https://www.design-get.com/color-2/)）

- **メインカラー**：ブランドの主役。全体の統一感をつくる土台。見出しの下線・セクションバー・重要リンク・強調に使う。
- **サブカラー**：メインを補完し、温度感やイメージを足す脇役。第二の強調（ステップ番号・小見出し・図の一部）に限定して使う。**メインより必ず控えめに。**
- **背景**：ベースは白。**セクションを 白 → wash → 白… と交互に塗り分ける**（メインカラーの極薄ティントを1つおきに敷く）。ページ全体を1色で塗らない／高彩度のベタ塗り背景は禁止。

```css
:root {
  --main:  #b04123;  /* メインカラー：ブランドの主役 */
  --sub:   #1c2951;  /* サブカラー：脇役。メインより控えめに */
  --wash:  #faf9f8;  /* メインの極薄ティント：交互背景に */
  --ink:   #141414;  /* メインテキスト */
  --gray:  #6b6b6b;  /* サブテキスト（白背景で 5.33:1） */
  --faint: #9a9a9a;  /* 装飾的な補助情報のみ（白背景で 2.81:1 → 4.5:1未達。意味のある情報には使わない） */
  --line:  #e6e6e6;  /* 細罫 */
  --line-strong: #111;
  --bg:    #ffffff;  /* 白（カード・ナビを浮かせる） */
  --accent: var(--main); /* 後方互換エイリアス */
}
```

**変えるのは `--main` `--sub` `--wash` の3変数だけ。他は触らない。**

### 面積比の目安（70 : 25 : 5）

- 白 ＋ wash …… 約70%（ベース。余白を機能させる）
- メインカラー …… 約25%（ポイント使い）
- サブカラー …… 約5%（さらに絞る）

### 印象からメインを決める（依頼者の言葉から）

「信頼感のある」「親しみやすい」など、感覚的な要望からメインカラーを引ける（クラウドソーシングの依頼イメージと同じ発想）。
**「あたたかい ⇄ つめたい」×「カジュアル ⇄ フォーマル」の2軸**で整理する。信頼と親しみ、高級と楽しいは対角の関係。

| 系統 | 出したい印象 | メインの方向 | 例 |
|---|---|---|---|
| あたたかい × カジュアル | 親しみ・あたたかい | オレンジ | `#ef7a45` |
| 〃 | 元気・活発 | 朱・バーミリオン | `#f24d29` |
| 〃 | 楽しい・明るい | イエロー | `#f5b301` |
| 〃 | やわらか・かわいい | ローズ・ピンク | `#e07a97` |
| つめたい × カジュアル | 清潔・さわやか | スカイブルー | `#2b9cf0` |
| 〃 | ナチュラル・やさしい | 葉のグリーン | `#6f8f4e` |
| あたたかい × フォーマル | 情熱・力強い | レッド | `#c0392b` |
| 〃 | 高級・上質 | 深いワイン・黒に近い色 | `#5a2a3a` |
| つめたい × フォーマル | 信頼・堅実 | 紺・ネイビー | `#1f4e79` |
| 〃 | 知的・クール | ティール | `#2f6f7a` |
| 〃 | 落ち着き・大人 | 深緑・スレート | `#3a5145` |
| 〃 | 上品・エレガント | すみれ | `#7c5aa6` |

メインが決まれば、サブと wash は次のように続ける。

### メインを決めたら、サブはこう選ぶ

1色決まれば、サブは狙いから逆算できる。迷ったら **同系のトーン違い** が最も外さない。

| 狙い・空気感 | サブの選び方 | 組み合わせ例（メイン → サブ） |
|---|---|---|
| まとまり・上品・静か | 同系のトーン違い（濃淡） | 深緑 → アイボリー |
| 調和・やわらかい | 色相環で隣の類似色 | 青 → 青緑 |
| 知的 × 温かさ | 寒色メイン × 暖色サブ | ネイビー → マスタード |
| きちんと × 親しみ | 落ち着き × 華やか | チャコール → コーラル |
| 信頼 × 前向き | 紺 × 黄緑 | 紺 → 黄緑 |
| 上品かわいい | 澄んだ色 × パステル | エメラルド → パステルピンク |

> サブは主張させない。候補が強すぎたら、彩度か明度を1段落として脇に回す。

### 極薄ティント（wash）のつくり方

メインカラーを HSL にして、**彩度 9〜15% / 明度 97〜98%** まで薄める（白に数%だけ混ぜた、うっすら色味を感じる程度）。

**使い方：ページ全体を wash で塗らない。** セクションを白と wash で交互に敷き、カード・ナビ・比較表は白（`--bg`）で抜くと、要素が上品に浮く。

```css
body { background: var(--bg); }                        /* ベースは白 */
section:nth-of-type(odd) { background: var(--wash); }  /* 1つおきに極薄ティント */
```

> ブラウザでメイン → サブ → wash を試せるツール：
> https://teppay0v0.github.io/hanabi-design-rules/color.html
> （メインを1色決めると、サブ候補と極薄背景を自動で提案する）

## タイポグラフィ

- **イタリック不採用**（設計的な使用も含めて完全に不使用）
- 強調は `font-weight` / `font-size` / `letter-spacing` のいずれか1つだけ
- 見出しは `clamp()` でフルワイド流体タイポグラフィ（`max-width` を設定しない）
- 日本語: Noto Sans JP / Noto Serif JP
- 欧文・数字: Inter

```css
h1 { font-size: clamp(40px, 8vw, 160px); font-weight: 800; letter-spacing: -.01em; }
h2 { font-size: clamp(32px, 5vw, 100px); font-weight: 800; letter-spacing: -.005em; }
```

## アクセシビリティの数値基準

引き算の美意識は、数値で裏付けられたときに初めて説得力を持つ。
[デジタル庁デザインシステム（DADS）](https://design.digital.go.jp/dads/resources/)を参考に、以下は必達ルールとして扱う。

| 項目 | 基準 |
|---|---|
| テキスト × 背景のコントラスト比 | **4.5:1 以上**（`--gray` `--faint` を薄くしすぎない） |
| 非テキスト要素（枠線・アイコン）× 背景 | **3:1 以上** |
| 本文・UI の文字サイズ | **16px 以上**を基準。14px 未満は原則使わない |
| 本文の行高 | フォントサイズの **1.5倍以上**（hanabiの 1.8〜2.0 はこの基準を満たす） |
| タップ・クリック領域 | **44px 角以上**。テキストリンクの見た目は小さくてよいが、`padding` でクリック可能領域を確保する |
| リンクの見分け方 | 色だけで区別しない。下線など色以外の装飾を必ず併用する |

> 出典: [カラー](https://design.digital.go.jp/dads/foundations/color/) / [タイポグラフィ](https://design.digital.go.jp/dads/foundations/typography/) / [リンクテキスト](https://design.digital.go.jp/dads/foundations/link-text/) / [ボタン](https://design.digital.go.jp/dads/components/button/)

**`--faint` の扱いに注意。** 白背景での実測は 2.81:1 で 4.5:1 に届かない。意味のある情報（読めないと困る注記・エラー文言など）には使わず、`--gray` 以上を使う。`--faint` は「読めなくても支障のない装飾的な補助情報」限定（例: セクション番号の添え字、タイムスタンプの補足）。細罫（`--line`）は装飾的な区切り線としての使用に限り、この数値基準の対象外とする。

## LP・HP のレイアウト原則

### 骨格
- 白基調（`#ffffff`）＋細罫（`1px solid var(--line)`）でセクションを分ける
- 全画面ステートメント（`min-height: 100vh`）を1箇所以上
- 横余白: `padding: 0 clamp(28px, 6vw, 120px)`
- 縦余白: `section { padding: 96px 0; }`（狭くしない）

### ナビゲーション
- 右側セクションドット（固定・小さい丸）でページ進捗を示す
- Sticky nav はシンプルに（ロゴ＋テキストリンクのみ）

### インタラクション（標準装備）
- スクロール出現アニメ（IntersectionObserver）
- 読書プログレスバー（上部 3px、アクセント色）
- ヒーロー入場アニメ（`@keyframes heroIn`）
- カスタムカーソル（◯リング）

### スナップスクロール（プロジェクター・プレゼン用）
```css
@media (min-width: 900px) {
  html { scroll-snap-type: y proximity; }
  .hero, section { scroll-snap-align: start; }
}
```

## スライドのルール

- 白基調＋細罫＋タイポグラフィのみでエディトリアル風に
- **1セクション1メッセージ**（`min-height: 100vh`）
- ベタ塗り・全面カラー背景は禁止
- 強調はサイズ差・ウェイト差・細罫の位置で

---

## 余白の考え方

> 余白は「何もない空間」ではなく「機能している空間」。
> 詰め込みすぎると何を見ていいか分からなくなる。
> 広く取りすぎかな？くらいがちょうどいい。

- `padding` は「窮屈かな？」と思ったら 2 割増す
- テキストの `line-height` は `1.8〜2.0`（日本語は特に広く。最低でもフォントサイズの1.5倍を割らない）
- 見出しに `max-width` を設定しない
- 要素が増えたら「削れないか」を先に考える
- 広く取ることを優先しつつ、使う余白の値には規則性を持たせる（行き当たりばったりの数値にしない）

## セルフチェックリスト

**制作後、必ず確認してから完成とする。**

```
□ 絵文字を使っていない
□ イタリックを使っていない
□ 高彩度のベタ塗り背景・グラデーションがない
□ 背景は白とwashの交互（全面を1色で塗らない）
□ ドロップシャドウがない
□ border-radius（角丸）がない
□ 色はメイン＋サブの 2 色まで（＋インク・グレー・罫）
□ サブカラーはメインより控えめに使っている
□ CTA はテキストリンクのみ（大型ボタンがない）
□ ストック写真・ヒーロー画像がない
□ 余白が十分に広い（詰まっていない）
□ 強調は 1 手段のみ
□ 見出しに max-width を設定していない
□ 意味のある情報は --gray 以上を使っている（--faint は装飾的な補助情報のみ）
□ 本文・UIの文字サイズが 16px 以上
□ リンク・CTAが下線など色以外の手段でも分かる
□ テキストリンクの実クリック領域が 44px 角相当を確保できている
```

## テンプレート集（スターター）の使い方

**白紙から書かない。** `references/` から用途に合うスターターを選んでコピーする。
依頼内容から業種が読み取れたら該当スターターを、当てはまらなければ汎用の `lp-starter.html` を使う。

| 用途・業種 | ファイル | 初期配色（main × sub） | 構成 |
|---|---|---|---|
| 汎用LP | `lp-starter.html` | 案件に合わせて変更 | ヒーロー／ステートメント／カード／比較表／ステップ／定義リスト／CTA |
| 飲食店 | `lp-restaurant.html` | テラコッタ × 深緑 | ヒーロー（本日の情報枠）／こだわり／おしながき（ドットリーダー）／コース／店舗情報／予約 |
| 美容室・サロン | `lp-salon.html` | ローズ × 深ローズ茶 | ヒーロー（空き情報枠）／コンセプト／メニュー料金／スタイリスト／初めての方へ／予約 |
| 士業・コンサル | `lp-professional.html` | 紺 × マスタード | ヒーロー（受付情報枠）／こんなときに／業務内容／料金／流れ／プロフィール／無料相談 |

### 手順

1. 該当スターターをプロジェクトフォルダにコピー
2. `:root` の `--main` `--sub` `--wash` をブランド色に変更（組み合わせは color.html で試せる）
3. サンプルコピーを案件の内容に差し替える（文体・構成はそのまま流用してよい）
4. 不要なセクションを削除する
5. セルフチェックリストで確認する

> 業種スターターのサンプル文は「架空店舗の完成例」なので、そのまま構成・トーンの手本として使える。
> 一覧デモ: https://teppay0v0.github.io/hanabi-design-rules/#templates
