# HANABI デザインルール

合同会社HANABI 廣田哲平が HP・LP・スライド制作で実践しているデザイン判断基準。
Claude Code が出力するデザインの質を上げるためのスキル。

## なぜこのスキルが必要か

Claude Code にデザインを任せると、デフォルトでは：
- 要素が詰まりがちになる
- カラフルで彩度が高くなる
- ドロップシャドウ・角丸・グラデーションが入る
- 絵文字がアイコン代わりになる
- ストック写真が使われる

これを防ぐための判断ルール集。**スターターテンプレートは `references/lp-starter.html`。白紙から書かない。**

---

## 根本哲学：引くこと

AIは「足す」のが得意。だから人が「引く」必要がある。

> 「AIが作るデザインって、なんか詰まってる感じがする。
> わたしが気をつけてるのは、むしろ引くこと。
> 余白って『何もない』んじゃなくて、ちゃんと機能してると思っていて。
> 詰め込みすぎると、見る人が疲れる。何を見たらいいか分からなくなる。
> だから意図的に『置かない』判断が大事。」

---

## 禁止事項（やらないこと）

以下は理由を問わず使用しない。

| 禁止 | 代わりに使うもの |
|------|----------------|
| 絵文字（すべての制作物） | モノライン系アイコン（ICON RAINBOW / ICONOON MONO） |
| イタリック | 使用しない。強調はサイズ・ウェイト・字間で |
| ベタ塗り背景・全面カラー背景 | 白（#ffffff）＋細罫（1px）のみ |
| ドロップシャドウ | 細罫（1px solid var(--line)）のみ |
| 角丸カード（border-radius） | 直角のみ |
| グラデーション | 単色のみ |
| 3色以上のカラーパレット | 白＋アクセント1色＋インク・グレー系のみ |
| 大型・角丸の CTA ボタン | テキストリンクのみ |
| ヒーロー写真・ストック画像 | 使用しない（タイポグラフィ・線・数字で構成） |
| 太字＋カラー＋italic の複合強調 | ウェイト・サイズ・letter-spacing のどれか1つだけ |
| 機械的な 8/16/24px 余白倍数 | 読みやすさ優先で広く取る |

## カラーパレット

```css
:root {
  --ink:         #141414;  /* メインテキスト */
  --gray:        #6b6b6b;  /* サブテキスト */
  --faint:       #9a9a9a;  /* 注記・ラベル */
  --line:        #e6e6e6;  /* 細罫 */
  --line-strong: #111;     /* 強調罫線 */
  --accent:      #ffe600;  /* アクセント ← ここだけ案件ブランド色に変更 */
  --bg:          #ffffff;  /* 背景 */
  --soft:        #f6f6f6;  /* 薄グレー（差し色） */
}
```

**`--accent` だけを案件ブランドカラーに変更する。他は変えない。**

---

## タイポグラフィ

- **イタリック不採用**（設計的な使用も含めて完全に不使用）
- 強調は `font-weight` / `font-size` / `letter-spacing` のいずれか1つだけ
- 見出しは `clamp()` でフルワイド流体タイポグラフィ（`max-width` を設定しない）
- 日本語: Noto Sans JP / Noto Serif JP
- 欧文・数字: Inter

```css
h1 { font-size: clamp(40px, 8vw, 160px); font-weight: 800; letter-spacing: -.01em; }
h2 { font-size: clamp(32px, 5vw, 100px); font-weight: 800; letter-spacing: -.005em; }
```

## LP・HP のレイアウト原則

### 骨格
- 白基調（`#ffffff`）＋細罫（`1px solid var(--line)`）でセクションを分ける
- 全画面ステートメント（`min-height: 100vh`）を1箇所以上
- 横余白: `padding: 0 clamp(28px, 6vw, 120px)`
- 縦余白: `section { padding: 96px 0; }`（狭くしない）

### ナビゲーション
- 右側セクションドット（固定・小さい丸）でページ進捗を示す
- Sticky nav はシンプルに（ロゴ＋テキストリンクのみ）

### インタラクション（標準装備）
- スクロール出現アニメ（IntersectionObserver）
- 読書プログレスバー（上部 3px、アクセント色）
- ヒーロー入場アニメ（`@keyframes heroIn`）
- カスタムカーソル（◯リング）

### スナップスクロール（プロジェクター・プレゼン用）
```css
@media (min-width: 900px) {
  html { scroll-snap-type: y proximity; }
  .hero, section { scroll-snap-align: start; }
}
```

## スライドのルール

- 白基調＋細罫＋タイポグラフィのみでエディトリアル風に
- **1セクション1メッセージ**（`min-height: 100vh`）
- ベタ塗り・全面カラー背景は禁止
- 強調はサイズ差・ウェイト差・細罫の位置で

---

## 余白の考え方

> 余白は「何もない空間」ではなく「機能している空間」。
> 詰め込みすぎると何を見ていいか分からなくなる。
> 広く取りすぎかな？くらいがちょうどいい。

- `padding` は「窮屈かな？」と思ったら 2 割増す
- テキストの `line-height` は `1.8〜2.0`（日本語は特に広く）
- 見出しに `max-width` を設定しない
- 要素が増えたら「削れないか」を先に考える

---

## セルフチェックリスト

**制作後、必ず確認してから完成とする。**

```
□ 絵文字を使っていない
□ イタリックを使っていない
□ ベタ塗り背景・グラデーションがない
□ ドロップシャドウがない
□ border-radius（角丸）がない
□ アクセント色は 1 色だけ
□ CTA はテキストリンクのみ（大型ボタンがない）
□ ストック写真・ヒーロー画像がない
□ 余白が十分に広い（詰まっていない）
□ 強調は 1 手段のみ
□ 見出しに max-width を設定していない
```

## スターターテンプレートの使い方

`references/lp-starter.html` を使う。**白紙から書かない。**

1. `lp-starter.html` をプロジェクトフォルダにコピー
2. `:root` の `--accent` をブランド色に変更（それだけでトーンが変わる）
3. テキストを差し替える（`{{TITLE}}` などのコメントを目印に）
4. 不要なセクションを削除する
5. セルフチェックリストで確認する
