# HANABI デザインルール

合同会社HANABI 廣田哲平が HP・LP・スライド制作で実践しているデザイン判断基準。
Claude Code が出力するデザインの質を上げるためのスキル。

## なぜこのスキルが必要か

Claude Code にデザインを任せると、デフォルトでは：
- 要素が詰まりがちになる
- カラフルで彩度が高くなる
- ドロップシャドウ・角丸・グラデーションが入る
- 絵文字がアイコン代わりになる
- ストック写真が使われる

これを防ぐための判断ルール集。**スターターテンプレートは `references/lp-starter.html`。白紙から書かない。**

---

## 根本哲学：引くこと

AIは「足す」のが得意。だから人が「引く」必要がある。

> 「AIが作るデザインって、なんか詰まってる感じがする。
> わたしが気をつけてるのは、むしろ引くこと。
> 余白って『何もない』んじゃなくて、ちゃんと機能してると思っていて。
> 詰め込みすぎると、見る人が疲れる。何を見たらいいか分からなくなる。
> だから意図的に『置かない』判断が大事。」

---

## 禁止事項（やらないこと）

以下は理由を問わず使用しない。

| 禁止 | 代わりに使うもの |
|------|----------------|
| 絵文字（すべての制作物） | モノライン系アイコン（ICON RAINBOW / ICONOON MONO） |
| イタリック | 使用しない。強調はサイズ・ウェイト・字間で |
| 高彩度のベタ塗り背景・全面カラー背景 | 白と極薄ティント（wash）を交互に |
| ドロップシャドウ | 細罫（1px solid var(--line)）のみ |
| 角丸カード（border-radius） | 直角のみ |
| グラデーション | 単色のみ |
| 主張する色を3色以上 | メイン＋サブの2色＋インク・グレー・罫のみ |
| 大型・角丸の CTA ボタン | テキストリンクのみ |
| ヒーロー写真・ストック画像 | 使用しない（タイポグラフィ・線・数字で構成） |
| 太字＋カラー＋italic の複合強調 | ウェイト・サイズ・letter-spacing のどれか1つだけ |
| 機械的な 8/16/24px 余白倍数 | 読みやすさ優先で広く取る |

## カラー設計（メイン＋サブの2色）

配色は **メインカラー1色 ＋ サブカラー1色** で組む。3色目・4色目を足さない。
（参照：[2色配色の考え方 — design-get.com](https://www.design-get.com/color-2/)）

- **メインカラー**：ブランドの主役。全体の統一感をつくる土台。見出しの下線・セクションバー・重要リンク・強調に使う。
- **サブカラー**：メインを補完し、温度感やイメージを足す脇役。第二の強調（ステップ番号・小見出し・図の一部）に限定して使う。**メインより必ず控えめに。**
- **背景**：ベースは白。**セクションを 白 → wash → 白… と交互に塗り分ける**（メインカラーの極薄ティントを1つおきに敷く）。ページ全体を1色で塗らない／高彩度のベタ塗り背景は禁止。

```css
:root {
  --main:  #b04123;  /* メインカラー：ブランドの主役 */
  --sub:   #1c2951;  /* サブカラー：脇役。メインより控えめに */
  --wash:  #faf2ef;  /* メインの極薄ティント：ページ背景・地に */
  --ink:   #141414;  /* メインテキスト */
  --gray:  #6b6b6b;  /* サブテキスト */
  --faint: #9a9a9a;  /* 注記・ラベル */
  --line:  #e6e6e6;  /* 細罫 */
  --line-strong: #111;
  --bg:    #ffffff;  /* 白（カード・ナビを浮かせる） */
  --accent: var(--main); /* 後方互換エイリアス */
}
```

**変えるのは `--main` `--sub` `--wash` の3変数だけ。他は触らない。**

### 面積比の目安（70 : 25 : 5）

- 白 ＋ wash …… 約70%（ベース。余白を機能させる）
- メインカラー …… 約25%（ポイント使い）
- サブカラー …… 約5%（さらに絞る）

### メインを決めたら、サブはこう選ぶ

1色決まれば、サブは狙いから逆算できる。迷ったら **同系のトーン違い** が最も外さない。

| 狙い・空気感 | サブの選び方 | 組み合わせ例（メイン → サブ） |
|---|---|---|
| まとまり・上品・静か | 同系のトーン違い（濃淡） | 深緑 → アイボリー |
| 調和・やわらかい | 色相環で隣の類似色 | 青 → 青緑 |
| 知的 × 温かさ | 寒色メイン × 暖色サブ | ネイビー → マスタード |
| きちんと × 親しみ | 落ち着き × 華やか | チャコール → コーラル |
| 信頼 × 前向き | 紺 × 黄緑 | 紺 → 黄緑 |
| 上品かわいい | 澄んだ色 × パステル | エメラルド → パステルピンク |

> サブは主張させない。候補が強すぎたら、彩度か明度を1段落として脇に回す。

### 極薄ティント（wash）のつくり方

メインカラーを HSL にして、**彩度 12〜24% / 明度 96〜98%** まで薄める（白に数%だけ混ぜた濃さ）。

**使い方：ページ全体を wash で塗らない。** セクションを白と wash で交互に敷き、カード・ナビ・比較表は白（`--bg`）で抜くと、要素が上品に浮く。

```css
body { background: var(--bg); }                        /* ベースは白 */
section:nth-of-type(odd) { background: var(--wash); }  /* 1つおきに極薄ティント */
```

> ブラウザでメイン → サブ → wash を試せるツール：
> https://teppay0v0.github.io/hanabi-design-rules/color.html
> （メインを1色決めると、サブ候補と極薄背景を自動で提案する）

## タイポグラフィ

- **イタリック不採用**（設計的な使用も含めて完全に不使用）
- 強調は `font-weight` / `font-size` / `letter-spacing` のいずれか1つだけ
- 見出しは `clamp()` でフルワイド流体タイポグラフィ（`max-width` を設定しない）
- 日本語: Noto Sans JP / Noto Serif JP
- 欧文・数字: Inter

```css
h1 { font-size: clamp(40px, 8vw, 160px); font-weight: 800; letter-spacing: -.01em; }
h2 { font-size: clamp(32px, 5vw, 100px); font-weight: 800; letter-spacing: -.005em; }
```

## LP・HP のレイアウト原則

### 骨格
- 白基調（`#ffffff`）＋細罫（`1px solid var(--line)`）でセクションを分ける
- 全画面ステートメント（`min-height: 100vh`）を1箇所以上
- 横余白: `padding: 0 clamp(28px, 6vw, 120px)`
- 縦余白: `section { padding: 96px 0; }`（狭くしない）

### ナビゲーション
- 右側セクションドット（固定・小さい丸）でページ進捗を示す
- Sticky nav はシンプルに（ロゴ＋テキストリンクのみ）

### インタラクション（標準装備）
- スクロール出現アニメ（IntersectionObserver）
- 読書プログレスバー（上部 3px、アクセント色）
- ヒーロー入場アニメ（`@keyframes heroIn`）
- カスタムカーソル（◯リング）

### スナップスクロール（プロジェクター・プレゼン用）
```css
@media (min-width: 900px) {
  html { scroll-snap-type: y proximity; }
  .hero, section { scroll-snap-align: start; }
}
```

## スライドのルール

- 白基調＋細罫＋タイポグラフィのみでエディトリアル風に
- **1セクション1メッセージ**（`min-height: 100vh`）
- ベタ塗り・全面カラー背景は禁止
- 強調はサイズ差・ウェイト差・細罫の位置で

---

## 余白の考え方

> 余白は「何もない空間」ではなく「機能している空間」。
> 詰め込みすぎると何を見ていいか分からなくなる。
> 広く取りすぎかな？くらいがちょうどいい。

- `padding` は「窮屈かな？」と思ったら 2 割増す
- テキストの `line-height` は `1.8〜2.0`（日本語は特に広く）
- 見出しに `max-width` を設定しない
- 要素が増えたら「削れないか」を先に考える

## セルフチェックリスト

**制作後、必ず確認してから完成とする。**

```
□ 絵文字を使っていない
□ イタリックを使っていない
□ 高彩度のベタ塗り背景・グラデーションがない
□ 背景は白とwashの交互（全面を1色で塗らない）
□ ドロップシャドウがない
□ border-radius（角丸）がない
□ 色はメイン＋サブの 2 色まで（＋インク・グレー・罫）
□ サブカラーはメインより控えめに使っている
□ CTA はテキストリンクのみ（大型ボタンがない）
□ ストック写真・ヒーロー画像がない
□ 余白が十分に広い（詰まっていない）
□ 強調は 1 手段のみ
□ 見出しに max-width を設定していない
```

## スターターテンプレートの使い方

`references/lp-starter.html` を使う。**白紙から書かない。**

1. `lp-starter.html` をプロジェクトフォルダにコピー
2. `:root` の `--accent` をブランド色に変更（それだけでトーンが変わる）
3. テキストを差し替える（`{{TITLE}}` などのコメントを目印に）
4. 不要なセクションを削除する
5. セルフチェックリストで確認する
