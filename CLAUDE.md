## 進め方
- ルールを変えるときは `SKILL.md` が正本。`references/` のテンプレートも整合させる。
- GitHub Pages デモ（teppay0v0.github.io/hanabi-design-rules/）に影響するため、公開前に見た目を確認。
- **公開リポジトリ**なので、機密・個人情報・未公開の事業情報は書かない。

## Git運用（神棚・厳守）
- **作業を始める前に必ず** `git pull --rebase` で最新を取り込む（mbp／スマホ／iMacで正本を共有しているため）。
- 区切りごとに `git add -A && git commit -m "…" && git push` で正本を更新する（こまめに）。
- コンフリクトしたら慌てず、差分を確認して手当てしてから続ける。
